## 구현 내용
1. **캠페인 즉시 마감 트리거**
   - `functions/src/campaigns.ts`에 `onCampaignUpdated` 함수 추가.
   - 캠페인 문서 업데이트 시, 지원자 수(`currentApplicants`)가 모집 인원(`recruitCount`)에 도달하면 즉시 캠페인 상태를 `selection_pending`으로 변경하고 텔레그램 마감 알림을 전송하도록 구현.

2. **자동 마감 스케줄러 (autoCampaignClose)**
   - 매일 자정(`0 0 * * *`) 실행.
   - 모집 중이면서 마감일(`deadline`)이 지난 캠페인을 조회하여 `selection_pending`으로 변경하고 텔레그램 마감 알림 전송.

3. **온보딩 넛지 스케줄러 (onboardingNudge)**
   - 매 2시간마다(`0 */2 * * *`) 실행.
   - 브랜드: 가입 후 48시간이 지나고 캠페인이 없는 경우 이메일 발송 ("첫 캠페인을 등록해보세요!").
   - 인플루언서: 가입 후 24시간이 지나고 팔로워가 없는 경우 이메일 발송 ("채널 연동만 하면 신청 가능해요!").
   - 인플루언서: 가입 후 48시간이 지나고 지원 내역이 없는 경우 이메일 발송 ("조건에 맞는 캠페인 N개가 기다리고 있어요!").
   - 중복 발송을 방지하기 위해 사용자 문서에 `sentNudge: true` 플래그를 추가.

4. **EmailJS REST API 연동**
   - Node.js 서버 환경에서 EmailJS로 메일을 보내기 위해 `functions/src/utils/emailjs.ts`를 신규 작성하여 HTTP 통신(Fetch API) 방식으로 구현.
