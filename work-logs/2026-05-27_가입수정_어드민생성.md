## 구현 내용
- **Firestore Security Rules 수정**: `crew_members` 누구나 등록(create) 및 어드민만 조회(read/write) 가능하도록 규칙을 수정하고 배포 완료하였습니다.
- **가입 폼 및 가입 페이지 수정**: `app/(auth)/join/page.tsx`와 `components/join/JoinComponent.tsx`를 정리하고, 최신 App Router 가이드라인에 맞춰 `app/join/page.tsx`에 `Suspense`를 씌우고 `components/JoinForm.tsx`에서 `addDoc`을 사용하여 직접 Firestore `crew_members`에 저장하도록 구현하였습니다. URL 파라미터 `type`과 `result`를 파싱하여 `userType`, `resultType`으로 함께 저장하도록 처리했습니다.
- **Firebase config 설정**: `next.config.js`에 공개 환경 변수(Firebase SDK keys)를 하드코딩 형태로 안전하게 주입해 빌드 시 정적 내보내기(`out/`) 상태에서도 Firebase SDK 초기화 및 Firestore 호출이 정상 작동하게 조치했습니다.
- **어드민 로그인 및 대시보드 구현**:
  - `/admin` 접속 시 `/admin/login`으로 자동 리다이렉트 처리.
  - `/admin/login`에서 간단 비밀번호(`anting2025admin`) 방식으로 로그인 처리 및 `sessionStorage` 세션 저장.
  - `app/admin/layout.tsx`에서 세션 유효성을 체크하여 비인증 유저 접근 제어.
  - `app/admin/dashboard/page.tsx`에서 Firestore `crew_members` 목록을 최신순으로 정렬하여 렌더링하고, KPI 카드(전체 크루, 광고주, 인플루언서 수), 실시간 이름/이메일 검색 및 유형별 필터링, 상세 정보 슬라이드오버(메모 수정/삭제 포함), CSV 내보내기(BOM 추가로 한글 깨짐 방지)를 구현했습니다.
  - Direct URL 접근 시 404가 발생하거나 메인 페이지로 리다이렉트되는 라우팅 이슈를 해결하기 위해, `firebase.json`에 `cleanUrls: true`와 `trailingSlash: true` 옵션을 지정하고 단일 페이지용 `rewrites` 설정을 제거하여 정적으로 빌드된 `/admin/login/index.html` 파일이 직접 서빙되도록 수정하였습니다.
